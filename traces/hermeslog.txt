    ###
    1. 專案目的:
    為國立臺南大學資訊工程學系制作一段60秒的形象宣傳影片，透過簡潔而有力的文字與視覺設計，向潛在學生成員及廣大公眾闡述信息工程學系的專長、成就及對臺灣數位經濟發展的貢獻。

    2. 目標觀眾:
    為本專案的主要目標客戶是資訊工程學系學生與其未來的師資，同時也著重設計吸引廣泛的公開教育及企業顧客群，如準備申請相關科目的民眾或一般想要了解大陸臺灣高等教育資源豐富度的人群。

    3. 核心訊息:
    - 國立臺南大學情報工程學系是一所專業而創新的學術機構，提供實踐性的學習環境與先進的研究基礎。
    - 本系具有強大的專欄領域知識與創新能力，特別是數位科技、人工智能與數據分析的領先研究。
    - 我們在教育及研究上的卓越，為臺灣的資訊產業發展做出了重要貢獻。
    - 加入資訊工程學系即加入一個充滿成就與創新的學習社群，一起塑造未來的技術方向。

    4. 每個 agent 的名稱及其 role、input、output、constraints：

    Planners Agent
    * Role: 負責整體計畫的策劃設計,包括影片主題及視覺框架。
    * Input: 專案目的、目標觀眾、核心訊息。
    * Output: 完成的計畫建議草案，包括影片主題的草圖與概念化的視覺藍圖。
    * Constraints: 核心資訊需符合影像創作者的角色與能力。在這個階段須保持開放性，讓方案可以靈活地調整與優化。

    Script Agent
    * Role: 撰寫宣傳影片劇本文字。
    * Input: Planners Agent 提供的影片主題草圖及視覺藍圖。
    * Output: 完成的宣傳影片文字劇本，包含對話、角色描述等信息。
    * Constraints: 正確反映目標觀眾的需求以及核心訊息。

    Visual Agents
    Vision Agent 1
    * Role: 根據 Script Agent 提供的文字劇本創作出視覺設計，例如圖像標誌等。
    * Input: 編劇輸出的宣傳影片文字劇本和 Vision Agent 2 可能需要提供的其他資源（如果有）。
    * Output: 完成的視覺設計元素，如標誌、配色方案等。
    * Constraints: 核心訊息以及目標觀眾要求在視覺設計中得到最佳實現。此外，也需要盡可能地遵守影片劇本中的文字提示和角色描述的要求。

    Vision Agent 2
    * Role: 負責宣傳片頭、節目間及結尾的特定視覺特效，與影片創作者分享劇本劇情以確保對標誌符的成功使用。
    * Input: Storyteller Agent 提供的文字劇本和 Viewers 目標觀眾相關資訊。
    * Output: 完成的特點視覺設計元素（如字體選擇、動畫特效等），用於確保宣傳片頭、結尾和整個節目都能實現故事敘事的需求。
    * Constraints: 视覚設計元素需符合目標觀眾的要求，並須保持核心訊息的一貫性。此外也需要遵循影片劇本的標誌符描述。

    Reviewer Agents
    Reviwer Agent 1
    * Role: 檢視 Visual Agent 轉換的電影腳本中的視覺設計是否正確符合劇情。
    * Input: Script 提供的文字劇本、Vision Agent 所提供的專屬視覺方案。
    * Output: 檔案檢查報告的撰寫，包括任何可能存在的問題以及改进建議。
    * Constraints: 驗證對比劇本中的角色描述和實際畫面之間是否存在差距或不一致的地方，這有助於確保宣傳影片能有效地達成目標。

    Reviewer Agent 2
    * Role: 檢視宣傳影片的整體設計方案，確保其符合目標觀眾並強調核心訊息。
    * Input: Planners Agent 提供的完整的計畫建議草案，包括文案劇本和 Vision Agent 輸出的電影視覺設計。
    * Output: 完成的審閱報告，包含對宣傳影片各部分的一般評估和改进建議。
    * Constraints：確保整個方案與目標觀眾有高度相關性及有效執行核心訊息。

    Finalizer Agent
    * Role: 汇總所有agents 規畫而成的計劃、審核意見後，確定最終版影片文案劇本並確認影片視覺設計的準備情況。
    * Input: Script Agent 提供的宣傳影片文字劇本文件；Reviewers Agent 提交的檔案檢查報告和方案建議草案；Vision Agents 區分提交的電影視覺設計元素及結案報告；其他需要整合的所有輸入資源（如果需要）。
    * Output: 完成的最終版宣傳影片文案劇本；以及視覺設計確認報告，包括視覺方案中的主要更新與修改說明。
    * Constraints：確保整體宣傳片主題、時長、目標觀眾與核心訊息一致。視覺設計需遵守所有先前意見和建議，以確保宣傳影片成功實現。

    調度員代理

    Script Agent
    * Role: 撰寫宣傳影片劇本文字。
    * Input: Planners Agent 提供的影片主題草圖及視覺藍圖。
    * Output: 完成的宣傳影片文字劇本，包含對話、角色描述等信息。
    * Constraints: 正確反映目標觀眾的需求以及核心訊息。

    Visual Agents
    Vision Agent 1
    * Role: 根據 Script Agent 提供的文字劇本創作出視覺設計，例如圖像標誌等。
    * Input: 編劇輸出的宣傳影片文字劇本和 Vision Agent 2 可能需要提供的其他資源（如果有）。
    * Output: 完成的視覺設計元素，如標誌、配色方案等。
    * Constraints: 核心訊息以及目標觀眾要求在視覺設計中得到最佳實現。此外，也需要盡可能地遵守影片劇本中的文字提示和角色描述的要求。

    Vision Agent 2
    * Role: 負責宣傳片頭、節目間及結尾的特定視覺特效，與影片創作者分享劇本劇情以確保對標誌符的成功使用。
    * Input: Storyteller Agent 提供的文字劇本和 Viewers 目標觀眾相關資訊。
    * Output: 完成的特點視覺設計元素（如字體選擇、動畫特效等），用於確保宣傳片頭、結尾和整個節目都能實現故事敘事的需求。
    * Constraints: 视覚設計元素需符合目標觀眾的要求，並須保持核心訊息的一貫性。此外也需要遵循影片劇本的標誌符描述。

    Reviewer Agents
    Reviwer Agent 1
    * Role: 檢視 Visual Agent 轉換的電影腳本中的視覺設計是否正確符合劇情。
    * Input: Script 提供的文字劇本、Vision Agent 所提供的專屬視覺方案。
    * Output: 檔案檢查報告的撰寫，包括任何可能存在的問題以及改进建議。
    * Constraints: 驗證對比劇本中的角色描述和實際畫面之間是否存在差距或不一致的地方，這有助於確保宣傳影片能有效地達成目標。

    Reviewer Agent 2
    * Role: 檢視宣傳影片的整體設計方案，確保其符合目標觀眾並強調核心訊息。
    * Input: Planners Agent 提供的完整的計畫建議草案，包括文案劇本和 Vision Agent 輸出的電影視覺設計。
    * Output: 完成的審閱報告，包含對宣傳影片各部分的一般評估和改进建議。
    * Constraints：確保整個方案與目標觀眾有高度相關性及有效執行核心訊息。

    Finalizer Agent
    * Role: 汇總所有agents 規畫而成的計劃、審核意見後，確定最終版影片文案劇本並確認影片視覺設計的準備情況。
    * Input: Script Agent 提供的宣傳影片文字劇本文件；Reviewers Agent 提交的檔案檢查報告和方案建議草案；Vision Agents 區分提交的電影視覺設計元素及結案報告；其他需要整合的所有輸入資源（如果需要）。
    * Output: 完成的最終版宣傳影片文案劇本；以及視覺設計確認報告，包括視覺方案中的主要更新與修改說明。
    * Constraints：確保整體宣傳片主題、時長、目標觀眾與核心訊息一致。視覺設計需遵守所有先前意見和建議，以確保宣傳影片成功實現。

    調度員代理

    Hermes Execution Trace

    - 平台：Hermes Agent
      - 使用了複雜的multi-agentworkflow來編排和執行一系列多個agent所必須完成的工作。
    - 動作模擬模式：Zero-paid-token mock/stub Dry-run, 模擬環境，不進行真實資料庫或雲端服務之數據使用行為。
    - 為什麼不使用paid API或是信用卡服務：在這個實驗階段的目的僅是測試和確認該multi-agentworkflow是否運作正常，所以沒有任何商業交易行為。在此過程中我們使用了一些mock/stub方式來模擬真實使用的API功能。

    - Steps
      - 步驟一（Coordinator Agent）：設定專案目的、目標觀眾等基本信息
      - 步驟二（Planner Agent）：根據提供的信息製作基本的大綱和視覺藍圖
      - 步驟三（Script Agent）：根據編劇的需求，撰寫影片的實際故事和字幕內容，確保所有的資訊都是完整的。
      - 步驟四（Visual Agent）：設計視頻素材及影像片段的拍攝計劃、並對拍攝的資料進行了整合及審查
      - 步驟五（Reviewer Agent）：檢查影片的整體狀態，包括每個scene的情節安排、影片內容等，確保所拍攝到的照片或影像符合預期標準。
      - 步驟六（Finalizer Agent）：根據各個步驟的結果整合所有信息，修改剪接計劃及其他相關文件。確保整齣影片能與整個流程保持一致。

    - Handoff 手續傳遞
      - 手順一 (Handoff / reviewer_feedback.json)：
        收集了所有的評分和建議意見並整合到一起，將最終的評價轉送到審查者，確認此阶段的工作。
      - 手順二 (Handoff finalizer_output.json)：整合所有前一步骤收集的信息，并与之前的手顺信息对比, 创建最终版本。

    - 所有最後提交的手順情報都明确注说明了final_video.mp4是由學生自發編輯完成。

    Baseline Comparison

    - Single-agent baseline:
      - 跟multi-agent workflow的主要異同點在於分工及控制，較多的細節由單一agents處理並完成相對較複雜的任務。

    - Multi-agent workflow:
      - 分工詳細，每個agent都扮演著不同的職責角色，能夠進行獨立的工作且無需專人監督所有步驟。
      - 適用在更廣泛的情況下使用，尤其適合處理多個agents需要共同完成複雜或大型的任務。
      - 很容易追蹤整個開發過程中的變化與細節。

    - 結構完整度
      - 多agent流程更加精確且明確地劃分出不同角色職責。比較單一agents則較為直接，但缺乏細節上的對照點。

    - 分工清楚度
      - 多agent流程下，每個agents之間的聯動更為密切。多個agents間存在著清晰、明確的工作分配及監督機制。
      - 单一agents沒有這種互相協調及配合的情況，但能更快的完成具體任務。

    - 可追蹤性
      - 多agent流程下，各個agents對工作的記錄及評估結果都能在系統中輕易訪問及追溯到。
      - 只有單一 agents 的環境會造成工作資料不易找尋及缺乏細節的記錄。
    - 版權與肖像權審查
      - 多agent流程中，版權和肖像權的相關問題是由所有agents共同審查並處理以確保沒有侵犯隱私或冒用標誌等行為。因此，審批過程會更嚴謹且有較高的透明度。

      - 使用僅於單一agents的情況下，可能會忽略某些文件的檢查程序以致可能出現版權風險。

    - 影片長度控制
      - 多agent流程中，每個agents的角色和職責均被優先確定和考慮到。以確保合適的影片長度和質量。這種方式提供了更多的調整空間來修正最終剪輯結果。
      - 個人使用的單一Agents處理該任務時通常會更加容易忽略控制影片長度的需求。

    - zero-paid-token 可完成性
      - 多agent流程中，在不使用paid或信用卡的情況下能夠輕鬆的驗證和測試任務是否可以成功完結。每一個agents在獨立工作時都可以通過零付token來補充其運作所需要的資源。

      - 使用單一agents的情況下，可能需要引入更複雜的方法與技術來實現相似的效果。

    - 最終影片製作可控性
      - 多agent流程中的可控性會有所改進。每個agents能夠分別完成自己的任務，並相給出最後的影片長度控制及審查建議。
      - 在進行單一agents的情況下，最終影片製作的掌控變得更繁雜。

    總之，使用多agent workflow在某些特定情況下是可以提高效率、減少錯誤，並能確保最終產品達到預期品質。因此，在未來，我們將進一步優化此流程，增加其可操作性與實用性。
